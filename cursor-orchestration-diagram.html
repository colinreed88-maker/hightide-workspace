What Wade does in the middle
1
Understands the request — translates plain English into the exact code changes needed
2
Breaks it into sub-tasks — e.g. "create page", "update nav", "add data query", in order
3
Sends each task to Cursor via ACP — Cursor executes them against the real codebase
4
Reviews the output — checks it looks right before anything gets pushed
5
Deploys — pushes to GitHub, Vercel picks it up, site updates live
6
Reports back — "Done. Here's what changed and the link."
