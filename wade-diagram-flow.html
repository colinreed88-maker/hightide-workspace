Wade × Cursor Agent Orchestration
How a single conversation becomes parallel coding work — coordinated and synthesized automatically
1
The request
Colin asks a question
A task comes in — Slack, Telegram, or the intranet. It could be a data question, a code change, a new dashboard page, or a diagnosis.
2
Orchestration
Wade plans the work
Wade reads the task, decides which parts need coding work, and prepares context packets — cursor rules, file paths, data schemas, relevant memories — for each sub-agent.
Main session · Full memory + data access
Context injected into each agent
Cursor Rules (.mdc)
Design system, page creation standards, BU mapping rules, Snowflake schema, property analysis conventions
design-system page-creation bu-filtering snowflake-schema property-pnl
Task Scope
Exact files to touch, which route to create, which data tables to use, what the output should look like
target file paths supabase tables acceptance criteria
Memory + Constraints
No npm, only pnpm. Negatives in parentheses. flow.life domain. Close status, data freshness, known quirks
pnpm only read-only Sage force-dynamic
3a
Cursor Agent
New page / route
  • Creates page.tsx, loading.tsx, content component
  • Applies design system + auth boilerplate
  • Registers in PAGE_REGISTRY
  • Writes data utility in lib/
3b
Cursor Agent
Data layer
  • Queries Supabase / Snowflake / Sage
  • Validates column names against schema ref
  • Writes typed interfaces
  • Tests query against real data
3c
Cursor Agent
Diagnosis / review
  • Reads error output or broken component
  • Checks cursor rules for constraints violated
  • Proposes fix with correct patterns
  • Returns a diff or replacement file
4
Synthesis
Wade reviews and delivers
Wade receives results from all agents, cross-checks them against constraints and memory, resolves any conflicts, and delivers a single coherent answer — code, analysis, or both — back to Colin.
5
Delivery
Colin gets a complete answer
Files committed, data confirmed, explanation written. One turn. Wade writes what happened to memory so the next session starts with full context.
The cursor rules — design system, page standards, Snowflake schema, BU mapping — travel with every agent spawn.
Agents never see MEMORY.md or personal context. They receive only the minimum needed to complete their task.
