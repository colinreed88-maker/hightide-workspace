## #openclaw-hangout Key Decisions (Mar 18–19)

### DeepSky Architecture
- **DeepSky** is Flow's core platform — a knowledge graph, not a relational DB. Built on Memgraph (graph DB) + NATS event streaming.
- One connected graph for operations AND financials. Every dollar knows why it was earned/spent.
- Uses "Primitives" (nodes) representing real things: buildings, units, people, agreements, time.
- **Smart Agreements (ACES V2):** Leases as executable code — WHO pays WHAT to WHOM under WHAT CONDITIONS at WHAT TIME. Phase 1 targeting end of March for Brickell sandbox.
- **DeepDive API:** `api.us.dev.deepsky.life/deepdive/v1/` — supports Cypher queries, MCP-compatible endpoint, text search. Auth via Flow JWT (staff scope).
- **NATS:** 69 context classes, 20 materialized in graph, 49 NATS-only. Agents can subscribe directly via `search_nats` / `list_nats_streams` MCP tools.
- **Waves** (operator UI) and **Lighthouse** (resident UI) are being wired up to DeepSky. Not fully ported yet — module by module.
- Pre-migration stack today: Yardi + Zendesk + Google Sheets. DeepSky migration is post-April.
- The "synthesis gap" is the core problem DeepSky solves: $100K+ water damage incident at a Flow property because signals existed in Yardi/Zendesk/Slack/work orders but were never stitched together.
- Full architecture detail: `memory/systems/deepsky-architecture.md` (authoritative — supersedes deepsky.md, deepsky-notes.md, deepsky-gap.md)

### MCP vs Skill Files Decision
- Cursor found "MCP + skills bundled together is much more powerful than MCPs alone" — independent convergence on OpenClaw's SKILL.md pattern.
- Decision: **Not competing — complementary.** Skills for information delivery/adaptive context; MCP for structured external tool integration.
- Key distinction: Skills are Markdown files (thin, flexible, ephemeral knowledge). MCP is a full protocol for connecting to external systems (deterministic, structured, isolated).
- For Flow's context: use MCP to connect to DeepSky, Yardi, Snowflake; use Skills for procedural knowledge (how to analyze a P&L, how to draft an email).
- MCP gained progressive discovery in Jan 2026, removing Skills' original context efficiency advantage.
- Full analysis: `memory/lessons/mcp-vs-skill-files.md` (authoritative — supersedes mcp-vs-skills.md)

### Multi-Claw Architecture Debate (Claw 25 + Uso + Beckett)
- Three topology patterns proposed for scaling to 600-unit buildings:
  1. **One Gateway per building** — isolation, separate config/state/workspace/ports
  2. **One Gateway, many agents** — route via `agents.list` + `bindings`
  3. **Orchestrator + workers** — front door claw dispatches via shared systems (work orders/DB/Slack)
- **Skills marketplace protocol** emerging: each claw broadcasts a `skills-manifest.json`, others self-select and bid on problems.
- Capability confidence scores (0–1), highest scorer acts, ties go to most recent.
- This belongs in OpenClaw repo as community protocol, NOT in Big Brain (Uso's workspace).
- Full notes: `memory/systems/openclaw.md`

## Cost Optimization — March 20, 2026 (afternoon)

### Changes Implemented (~1:21 PM UTC)
Prompted by Colin's question about token spend in #openclaw-hangout. Implemented the following config changes:

**Config (openclaw.json):**
- Context TTL: 1h → 20min
- Session idle timeout: 240min → 60min
- Heartbeat: 30min → 45min
- Sub-agent max concurrent: 8 → 4
- Children per agent: 4 → 2

**Cron changes:**
- Disabled "Overnight Learning" cron — was firing every hour, 25 min/run, 12 consecutive errors. Single biggest cost driver.
- Added "Daily Spend Report" cron at 11pm ET → both Telegram and Slack (#openclaw-hangout)
- Routine crons (calendar sync, RSS, Granola, daily digest) → Haiku model
- Analysis crons (morning brief, weekly review, spend report) → stay on Sonnet

**Model routing:**
- Sub-agents → `claude-haiku-4-5-20251001` (~20x cheaper per token)
- Compaction → Haiku

**Context:** Today's spend was $8.01 by 1pm UTC (vs $0.06 on Mar 18). Overnight Learning job alone likely drove most of that. Expected tomorrow's spend to be well under $20 with these changes.
